## v1.2.0 (2026-02-10)

Полная переработка архитектуры: от блоков с типами длительности к помодоро-циклам как центральной механике.

**Новое:**
- Помодоро — автоматические циклы 25+5 мин весь рабочий день, каждый 4-й — длинный перерыв
- События (созвоны, встречи) — отдельная сущность, не задача, не в статистике; во время события помодоро молчит
- Статусы задач: Grooming → В работе → Заблокировано → Готово; смена из бота и Web App
- Описание и ссылка у задач — показываются при выборе в помодоро
- 40+ спам-текстов с разными сценариями
- Итог дня: помодоро-статистика + перенос незавершённых задач на следующий рабочий день
- `/settings` объединяет /stop, /pause, /resume, /admin через inline-кнопки
- Авторассылка changelog при деплое (`version_notify.py`)

**Удалено:**
- Тихое время, минимальное время задачи, группировка блоков
- Команда `/next` (заменена навигацией в `/plan`)

## История разработки (архив батчей)

Внутренние заметки о реализации. Актуальны как справка по тому, что и когда менялось в коде.

### Этапы 1-7: Базовая реализация

**Этап 1:** Скелет — `.env.example`, `config.py`, модели БД (9 таблиц), WAL + FK для SQLite

**Этап 2:** CRUD + FastAPI — `db/crud/`, `api/deps.py`, Pydantic v2 схемы, routes users/categories

**Этап 3:** Скелет бота — Bot + Dispatcher, middleware whitelist, `/start`, `/help`, `/admin`, polling/webhook

**Этап 4:** Полный API — tasks, blocks (auto-distribute, carry-over, overlap warnings), schedule, stats, goals

**Этап 5:** Команды бота — `/stop`, `/pause`, `/resume`, `/settings`, `/stats`

**Этап 6:** Фронтенд скелет + экраны 0–3 — React + TS + Vite, Zustand, тёмная тема, TimezoneSelect, Categories, WeekSchedule, Goals

**Этап 7:** Фронтенд экраны 4–6 — Backlog, Calendar (drag & drop, @dnd-kit, 3 типа блоков), Summary (recharts), Settings

### Batch 1–3: UX-фиксы

- Batch 1: фиксы форм, валидация, пустые состояния
- Batch 2: фиксы drag & drop, overlap-предупреждения, поля задач
- Batch 3: фиксы автораспределения, поддержка эпиков

### Batch 3.1: Дополнительные фичи

- Имя блока с эмодзи при автогруппировке (например, «💼 Работа» вместо «Работа»)
- Прочие UX-улучшения автодистрибуции

### Batch 4: Мульти-задачи + эпик-drag

- `allow_multi_per_block`, `device_type`, `preferred_time` (впоследствии удалены в v1.5.1)
- Drag эпиков в календаре

### Batch 5–6: UX-фиксы

- Batch 5: 5 исправлений UI и поведения
- Batch 6: 3 исправления

### Batch 7–7.3: Scheduler + напоминания + callback-хэндлеры

- Batch 7: APScheduler, восстановление при рестарте, все типы напоминаний, спам-машина
- Batch 7.1: пост-фиксы scheduler
- Batch 7.2: команды `/plan` и `/next`
- Batch 7.3: админ-панель по username + кнопки выбора

### Batch 8–11: UX и новые фичи

- Batch 8: 5 исправлений
- Batch 9: сортировки, группировки, табличный вид в бэклоге
- Batch 10: Access Request, Timezone Picker
- Batch 11: спам для пустых слотов (впоследствии удалён)

### Batch 12: Деплой

- `nginx/task-planner.conf` — reverse proxy, SSL
- `systemd/task-planner.service`
- `setup.sh` — установочный скрипт
